# Copilot Instructions

Welcome to our project! Here are some guidelines for working with GitHub Copilot:

## General Guidelines

1. **Be Specific**: Write clear, descriptive comments to guide Copilot
2. **Review Suggestions**: Always review and test Copilot's suggestions
3. **Maintain Style**: Follow our project's coding conventions

## Best Practices

- Use descriptive function and variable names
- Break down complex tasks into smaller steps
- Include type hints where applicable
- Write comprehensive docstrings

## Project-Specific Patterns

- Use `snake_case` for variable and function names
- Follow our established error handling patterns
- Maintain consistent logging practices

## When to Use Copilot

- Generating boilerplate code
- Writing test cases
- Creating documentation
- Implementing common algorithms

## When to Avoid Copilot

- Security-sensitive code
- Business-critical logic
- Complex architectural decisions

## Feedback

If you notice any issues with Copilot's suggestions:

1. Report them in #copilot-feedback
2. Include the context and problematic suggestion
3. Suggest improvements if possible

Happy coding!
